## 1. 设计图

图稿改动集中在左侧顶部绿色区域：

- 中间用环形图显示今日挂号总数及“待接诊 / 接诊中”占比。
- 右侧用横向条形图显示各挂号时段人数。

说明：设计图由图片模型在原图基础上局部生成，主要用于确认布局、比例、图表类型和色彩。模型可能轻微重绘原图中的小字号文字；实际开发必须使用现有 Vue 组件和真实业务文案，不能直接照搬图中的小字。

设计图中的头像、职称和医院名称目前没有稳定接口字段，因此阶段 A 未伪造这些内容；后续只有在医生档案接口补齐后才接入。

## 2. 数据口径

第一阶段只使用 `GET /api/v1/patient/doctor/{employee_uuid}/queue` 已有数据，不新增后端接口，不展示无法验证的统计。

| 可视化 | 数据来源 | 计算方式 |
| --- | --- | --- |
| 今日挂号总数 | 队列数组 | `queueItems.length` |
| 待接诊 | `visit_state` | 统计已挂号、待接诊状态 |
| 接诊中 | `visit_state` | 统计接诊中状态 |
| 分时段挂号 | `time_range` | 按挂号时段分组计数 |
| 最后刷新 | 前端请求完成时间 | 每次成功加载后更新时间 |

第一阶段不展示“已完成接诊”“平均等待时长”“超时患者”“接诊趋势”，因为当前队列接口不直接提供这些数据。

## 3. 页面结构

```text
┌──────────────────── 医生工作台 / 挂号总览 ────────────────────┐
│ 医生身份                  │ 今日挂号状态       │ 分时段挂号       │
│ 李沐川 / 主治医师          │       12           │ 09:00 ▇▇  2    │
│ 神经内科门诊              │   [环形占比图]      │ 09:20 ▇▇  2    │
│ 上海市第一人民医院         │ 待接诊 10          │ 09:40 ▇   1    │
│ 最后刷新 09:32            │ 接诊中  2          │ 10:00 ▇▇▇ 3   │
└──────────────────────────────────────────────────────────────┘
```

环形图使用青绿色表示待接诊、暖橙色表示接诊中，同时保留文字图例，避免只依赖颜色传达状态。条形图按时间升序排列，不按人数排序；条尾显示准确人数，条长只承担快速比较作用。

## 5. 状态与响应式

- 加载中：显示轻量占位，不先显示 0，避免误判为空队列。
- 空队列：环图中心显示 0，条形图区显示“今日暂无挂号时段”。
- 时段缺失：统一归入“时间待确认”，不能静默丢弃。
- 单一状态：环图显示完整单色圆环，并保留 0 值图例。
- 宽度不足：1180px 以下三段布局变为单列，图表标签不得被截断。
- 可访问性：状态同时使用颜色、文字和数字；正文及图例满足基础对比度要求。

## 6. 验收标准

1. 环形图总数、两类状态之和与下方队列数据一致。
2. 各时段条形图人数之和与今日挂号总数一致，包括“时间待确认”。
3. 开始接诊后，待接诊与接诊中数量在接口刷新后同步变化。
4. 15 秒轮询不会造成图表闪烁、布局跳动或重复定时器。
5. 空队列和单一状态下不出现 `NaN`、除零错误或错误比例。
6. 设计改动不影响叫下一位、开始接诊、继续接诊和进入接诊详情。
7. 桌面端与窄屏均保持中文标签完整、图例清晰、按钮可操作。
