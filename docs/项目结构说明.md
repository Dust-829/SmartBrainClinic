## 1. 仓库顶层结构

### 1.1 顶层目录

- `.agents/`：记录后续 agent 协作约束，当前最关键文件是 `.agents/AGENTS.md`。
- `.claude/`：项目内安装的 skill 资源目录，当前主要是 `impeccable` 前端/UI 相关技能脚本。
- `.codex/`：Codex 项目级辅助目录。
- `.idea/`：IDE 工程配置。
- `.pytest_cache/`：pytest 缓存目录。
- `backend/`：FastAPI 微服务后端主目录，是当前后端真实实现所在位置。
- `database/`：数据库相关资源目录。
- `docs/`：项目规划、回顾、问题记录、架构说明和设计资产。
- `frontend/`：Vue 3 + Vite 前端工程，当前已实现患者端主链路并保留医生端/管理员端占位结构。
- `outputs/`：输出产物目录。
- `scripts/`：项目辅助脚本目录。

### 1.2 顶层文件

- `.gitattributes`：Git 属性配置。
- `.gitignore`：Git 忽略规则。
- `PRODUCT.md`：产品上下文文件，给设计和 agent 说明产品定位、用户、风格和边界。
- `README.md`：仓库总说明，但其中部分前端状态描述已经落后于当前实际代码。

## 2. 当前前端结构

前端当前已经是一个可运行的 Vue 3 工程，不是纯规划状态。现状可以概括为：

- 技术栈：Vue 3 + Vite + Vue Router + Pinia + Axios + Element Plus。
- 路由结构：一个工程中放三端路径。
- 当前已落地最完整的是患者端。
- 医生端和管理员端目前主要还是最小占位结构。

### 2.1 `frontend/` 顶层文件

- `frontend/package.json`：前端依赖和脚本入口。
- `frontend/pnpm-lock.yaml`：pnpm 锁文件。
- `frontend/pnpm-workspace.yaml`：pnpm workspace 配置。
- `frontend/index.html`：Vite 前端 HTML 容器。
- `frontend/vite.config.ts`：Vite 构建配置。
- `frontend/tsconfig.json`：TypeScript 主配置。
- `frontend/tsconfig.node.json`：Node/Vite 侧 TS 配置。

### 2.2 `frontend/src/` 根文件

- `frontend/src/main.ts`：前端应用启动入口，挂载 Pinia、Router、Element Plus。
- `frontend/src/App.vue`：应用外层壳，当前挂了一个全局 `RoleSwitcher`，但它只是开发调试工具，不是正式入口。
- `frontend/src/env.d.ts`：Vite 环境变量类型声明。

### 2.3 路由与布局

- `frontend/src/router/index.ts`：当前前端总路由表。
  - `/patient/*`：患者端主链路。
  - `/doctor`：医生端占位入口。
  - `/admin`：管理员端占位入口。
- `frontend/src/layouts/PatientLayout.vue`：患者端主布局，移动端优先，控制沉浸式页面和头部区域。
- `frontend/src/layouts/DoctorLayout.vue`：医生端布局，占位版桌面端工作台外壳。
- `frontend/src/layouts/AdminLayout.vue`：管理员端布局，占位版桌面后台外壳。

### 2.4 API 与状态

- `frontend/src/api/http.ts`：Axios 实例和统一错误提示逻辑，默认走 `VITE_API_BASE_URL` 或 `http://localhost:8000` 网关。

## 3. 当前后端整体结构

后端真实实现位于 `backend/`，采用 FastAPI 微服务结构。

### 3.1 当前微服务拆分

- Gateway：`backend/app/main.py`，端口 `8000`，统一入口和转发。
- Auth：`backend/app/microservices/auth/`，端口 `8001`，当前更像基础资料/员工资料服务。
- Patient：`backend/app/microservices/patient/`，端口 `8002`，患者建档、AI 分诊、医生推荐、挂号、候诊。
- Medical：`backend/app/microservices/medical/`，端口 `8003`，病历、检查检验、接诊、影像辅助、医生 AI 助手。
- Pharmacy：`backend/app/microservices/pharmacy/`，端口 `8004`，处方、发药、退药、库存。
- Billing：`backend/app/microservices/billing/`，端口 `8005`，支付、账单、退费和退款保护。

### 3.2 公共后端基础模块

- `backend/app/common/`：微服务共用配置、数据库、MQ、Nacos、AI 客户端、AI 审计、统一响应等基础能力。

- `backend/app/main.py`：Gateway FastAPI 应用入口。
  - 暴露 `/api/v1/{service}` 统一代理入口。

  - 服务发现失败时回退到本地默认端口。
  - 当前已加 CORS，允许前端 `5173` 端口访问。

## 12. 当前结构层面的真实结论

### 12.1 项目已经不是“只有后端”

虽然 `README.md` 里还残留“前端尚未初始化正式 Vue 3 工程”的表述，但真实代码已经是：

- 后端微服务可运行。
- 前端 Vue 工程已存在。
- 患者端主链路已是当前实现最完整的部分。

### 12.2 Auth 服务当前更像“基础资料服务”

从现有目录和职责来看，`auth` 这个名字容易让人误解为“完整认证中心”，但目前它更多承载：

- 医生、员工、科室、诊室等基础数据。
- 医生向量、擅长方向等辅助能力。

也就是说，它现在并不是一个完整成熟的三端统一登录中心。

### 12.3 前端三端独立方向已经落在结构上，但还没落在登录和守卫上

当前结构已经有：

- `PatientLayout`
- `DoctorLayout`
- `AdminLayout`
- `/patient`
- `/doctor`
- `/admin`

但还没有完全落成：

- 医生端独立登录页
- 管理员端独立登录页
- 独立会话状态
- 独立路由守卫

### 12.4 AI 相关代码已经分散进入多个服务

目前仓库里 AI 不只在一个地方：

- `patient/services/ai_triage.py`：患者 AI 问诊。
- `patient/services/ai_scheduling.py`：AI 排班辅助。
- `medical/services/ai_draft.py`：AI 病历草稿。
- `medical/services/ai_image_inference.py`：AI 影像辅助。
- `pharmacy/services/ai_prescription.py`：AI 处方建议。
- `common/ai_*`：通用模型调用、审计、校验、嵌入和复核。
