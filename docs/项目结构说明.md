# 项目结构说明

更新时间：2026-07-10。

本文档用于快速定位 SmartBrainClinic 的运行入口、业务模块和常用文件，不逐个罗列所有源码文件。总体状态见 [项目规划.md](./项目规划.md)，前端执行状态见 [frontend-plan.md](./frontend-plan.md)。

## 1. 仓库顶层

```text
SmartBrainClinic/
├─ backend/              FastAPI 微服务、迁移、脚本和测试
├─ frontend/             Vue 3 前端
├─ docs/                 长期维护文档与设计资产
├─ PRODUCT.md            产品与设计原则
├─ README.md             启动说明和文档入口
├─ .agents/AGENTS.md     仓库内 agent 约束
└─ .claude/skills/       项目内设计技能
```

早期 `services/` 等占位内容不是当前运行主线，判断实现时以 `backend/` 和 `frontend/` 为准。

## 2. 前端结构

```text
frontend/src/
├─ api/                  Axios 封装和领域 API
├─ components/           公共组件与患者端组件
├─ layouts/              Patient / Doctor / Admin 三端布局
├─ router/               路由表和 session 守卫
├─ stores/               三端 session 与患者业务状态
├─ styles/               全局样式
├─ views/
│  ├─ patient/           患者页面
│  ├─ doctor/            医生登录、工作台、接诊页
│  └─ admin/             管理员登录和控制台
├─ App.vue               根 router-view
└─ main.ts               前端入口
```

### 2.1 路由与布局

- `frontend/src/router/index.ts`：三端路由表。
- `frontend/src/router/sessionGuard.ts`：角色登录态判定和跳转。
- `frontend/src/layouts/PatientLayout.vue`：患者端移动布局。
- `frontend/src/layouts/DoctorLayout.vue`：医生桌面工作台布局。
- `frontend/src/layouts/AdminLayout.vue`：管理员后台布局。
- `frontend/src/views/PortalEntryView.vue`：根入口页。

### 2.2 状态

- `patientSession.ts`、`doctorSession.ts`、`adminSession.ts`：三端独立身份。
- `patientFlow.ts`：患者单次挂号流程和 AI 分诊会话。
- `patientRegisterHistory.ts`：本地待合并的挂号历史 60 秒共享缓存。
- `sessionStorage.ts`：本地 session 序列化工具。

### 2.3 医生端可视化

- `DoctorWorkbenchView.vue`：接口加载、轮询、队列行为和页面组合。
- `doctor/components/DoctorQueueDonut.vue`：SVG 状态环图。
- `doctor/components/DoctorQueueTimeBuckets.vue`：CSS 分时段条形图。
- `DoctorEncounterView.vue`：病历、AI 辅助和开单主流程。

## 3. 后端结构

```text
backend/
├─ app/
│  ├─ main.py                         Gateway
│  ├─ common/                         配置、HTTP 客户端、AI 会话等共享能力
│  └─ microservices/
│     ├─ auth/                         基础资料服务
│     ├─ patient/                      患者、分诊、挂号、排班、候诊
│     ├─ medical/                      病历、开单、AI 辅助
│     ├─ pharmacy/                     药品和处方
│     └─ billing/                      收费和退费
├─ migrations/                         增量 SQL
├─ scripts/                            演示数据和维护脚本
├─ tests/                              后端测试
├─ his_db_backup.sql                   基础库与演示数据备份
└─ docker-compose.yml                  本地基础设施
```

### 3.1 Gateway 与共享能力

- `backend/app/main.py`：根据 `/api/v1/{service}` 转发请求。
- `backend/app/common/clients.py`：本地待合并的共享 `httpx.AsyncClient` 与服务客户端。
- `backend/app/common/config.py`：通用配置。
- `backend/app/common/ai_conversation.py`：AI 会话模型与读写 helper。
- `backend/app/common/ai_governance.py`：AI 来源、审计和治理能力。
- `backend/app/common/idempotency.py`、`outbox.py`：高风险写操作与事件可靠性基础。

### 3.2 Auth

- API：`microservices/auth/api/auth.py`
- 服务层：`microservices/auth/services/auth_service.py`
- 模型：`microservices/auth/models/`

主要承载科室、医生、诊室、挂号级别和结算类别等基础资料。当前更接近基础资料服务，不是完整账号认证中心。

### 3.3 Patient

- API：`microservices/patient/api/patient.py`
- 核心服务：`microservices/patient/services/patient_service.py`
- AI 分诊：`microservices/patient/services/ai_triage.py`
- 模型：`microservices/patient/models/`

主链覆盖患者建档、AI 分诊、医生推荐、排班、挂号、支付、候诊和医生队列。

### 3.4 Medical

- API：`microservices/medical/api/medical.py`
- AI 草稿：`microservices/medical/services/ai_draft.py`
- 相似病历与助手：`microservices/medical/services/`
- 挂号事件消费者：`microservices/medical/workers/register_consumer.py`
- 医生 Agent：`microservices/medical/services/agent/`

主链覆盖病历草稿、医生确认、检查 / 检验 / 处置、相似病例和医生 AI 辅助。

### 3.5 Pharmacy 与 Billing

- Pharmacy：药品字典、库存、处方推荐、发药和退药。
- Billing：账单、支付、收费防重和退费 Saga。

## 4. 数据库与迁移

- 基础库：`backend/his_db_backup.sql`。
- 增量迁移：`backend/migrations/*.sql`，按文件名顺序执行。
- AI 会话迁移：`20260708_01_create_ai_conversation_tables.sql`。
- 当前实例关系说明：[数据库表结构说明.md](./数据库表结构说明.md)。

代码模型、迁移文件和实际数据库实例可能处于不同状态，排查问题时三者必须分别核对。

## 5. 测试与演示

- 后端测试：`backend/tests/`。
- 全链路演示数据：`backend/scripts/seed_full_demo_data.sql`。
- 演示数据说明：[全链路演示数据清单.md](./全链路演示数据清单.md)。
- 前端验证：在 `frontend/` 下运行 `npm run build`，并执行关键路由浏览器回归。

## 6. 常用定位

| 目标 | 首选文件 |
| --- | --- |
| 改三端入口或守卫 | `frontend/src/router/index.ts`、`sessionGuard.ts` |
| 改患者挂号流程 | `patientFlow.ts` 和患者端 views |
| 改医生候诊工作台 | `DoctorWorkbenchView.vue` |
| 改医生接诊 / 开单 | `DoctorEncounterView.vue`、`api/medical.ts` |
| 改挂号 / 候诊后端 | `patient_service.py` |
| 改 AI 病历上下文 | `register_consumer.py`、`ai_draft.py`、`ai_conversation.py` |
| 改跨服务 HTTP 调用 | `backend/app/common/clients.py` |
| 改收费 / 退费 | Billing service 与对应测试 |

## 7. 文档边界

`docs/` 同时维护两类资料：

- 当前依据：总体规划、前端计划、结构说明、数据库说明、演示数据和协作规范。
- 历史与报告留档：阶段回顾、问题记录、设计图和已完成专项方案。

已完成专项的当前结论应并回主文档；设计、验收和问题复盘在标注日期与适用范围后保留，便于报告引用与过程追溯。
