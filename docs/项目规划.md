# 项目规划

## 阶段一：项目启动

- 建立 GitHub 仓库与团队协作规则。
- 整理需求文档、模块边界和技术路线。
- 明确每位组员负责的端或服务。
- 建立前端、后端、AI 服务和数据库的基础工程。

## 阶段二：基础业务闭环

- 患者注册登录、智能问诊、挂号支付模拟。
- 医生叫号接诊、病历初稿、检查检验申请。
- 药房发药、费用记录、基础统计。
- PostgreSQL 核心表结构设计与初始化脚本。

## 阶段三：AI 能力接入

- LLM 智能问诊与病历初稿生成。
- Top-3 / Top-5 医生推荐。
- AI 检查检验建议。
- AI 处方推荐，但所有医疗结论必须由医生确认后生效。

## 阶段四：医学影像算法

- 影像上传与结果回传接口。
- 脑部 CT / MRI 肿瘤识别算法服务。
- 影像识别概率、位置和辅助报告展示。
- 医生端结果确认流程。

## 阶段五：优化与验收

- 权限、日志、异常处理和接口文档完善。
- 前端页面一致性和交互优化。
- 业务流程联调。
- 项目演示材料和验收文档整理。

## 推荐分工

- 前端组：Vue 3 页面、路由、状态管理和 API 对接。
- 后端业务组：门诊、药房、医技、管理端服务。
- AI 服务组：LLM 调用、提示词、问诊与推荐能力。
- 算法组：医学影像模型、推理接口和结果可视化数据。
- 数据库与 DevOps：表结构、初始化脚本、部署和 GitHub 工作流。
