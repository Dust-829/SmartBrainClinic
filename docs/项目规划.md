# 项目规划

本文档以当前仓库的实际实现为基线：后端主实现为 Python FastAPI 微服务，前端仍待初始化，`services/` 目录中的 Java 服务仅为早期规划占位。

## 当前基线

已完成或已有代码基础：

- FastAPI 微服务框架：Gateway、Auth、Patient、Medical、Pharmacy、Billing。
- PostgreSQL + SQLModel/SQLAlchemy Async 数据访问。
- pgvector 向量字段与医生/药品相似度检索能力。
- RabbitMQ 消息队列与 Outbox 扫描任务。
- Nacos 服务发现 fallback 机制。
- Redis 依赖预留。
- LLM 能力：AI 分诊、医生推荐、病历草稿、处方推荐、排班微调、医生评价。
- Docker Compose 基础设施编排。

当前主要缺口：

- 前端工程尚未开始。
- 文档、配置命名和当前实现曾存在不一致，正在逐步修正。
- 缺少自动化测试与 CI。
- 鉴权、权限控制和审计日志尚不完整。
- 容器内 Nacos 注册地址需要生产化修正。
- 配置安全需要加强，真实密钥必须移出代码。

## 阶段一：工程基线修正

目标：让项目可被新成员准确理解、可本地启动、可安全继续开发。

- 更新 README、规划文档和架构说明，统一 FastAPI 微服务路线。
- 新增 `.env.example`，明确所有必要环境变量。
- 移除代码中的真实 LLM API Key，并轮换已暴露密钥。
- 统一服务命名：特别是 `bill` / `billing` 的路由、配置和文档命名。
- 修正 Docker 环境下 Nacos 注册地址，避免容器内使用 `127.0.0.1` 注册。
- 明确数据库初始化方式：使用现有 SQL 备份，或补迁移工具方案。

## 阶段二：最小业务闭环

目标：形成可演示的门诊全链路。

- 患者建档与查询。
- AI 分诊，多轮问诊生成症状摘要和推荐科室。
- 医生推荐，结合医生擅长、向量相似度和有效排班。
- 患者选择号源并挂号。
- 挂号缴费和状态激活。
- 医生端查看候诊队列并接诊。
- AI 病历初稿生成。
- 医生确认诊断、开立处方。
- 患者缴费，药房发药，库存扣减。

验收标准：

- 以上流程可通过 API 连续跑通。
- 每个服务 `/health` 正常。
- 关键异常可返回明确错误信息。

## 阶段三：测试与稳定性

目标：降低复杂业务回归风险。

- 补充单元测试：业务服务函数、状态流转、金额计算、库存扣减。
- 补充集成测试：挂号、缴费、退费、发药、退药。
- 补充并发测试：同一号源抢号、重复挂号、库存不足。
- 补充消息测试：Outbox 投递、消费者幂等、失败重试。
- 建立 GitHub Actions：安装依赖、静态编译、测试、基础 lint。
- 为后台任务增加可观测日志和异常隔离。

## 阶段四：前端初始化

目标：先做可操作工作台，不做展示型首页。

建议技术栈：

- Vue 3
- Vite
- TypeScript
- Element Plus
- Pinia
- Axios

建议优先页面：

- 患者端：建档、AI 分诊、医生推荐、号源选择、我的挂号。
- 医生端：候诊队列、病历确认、检查/处方开立。
- 药房端：待发药列表、发药、退药。
- 财务端：待缴费列表、缴费、退费。
- 管理端：排班规则、排班申请审批、实际排班干预。

## 阶段五：AI 与医学影像能力完善

目标：把 AI 从“调用接口”推进到“可控辅助决策”。

- 标准化 AI 输入输出结构，避免业务层解析自然语言。
- 为 AI 分诊、病历草稿、处方推荐增加医生确认状态。
- 为处方推荐增加禁忌、过敏史、库存和价格约束。
- 医学影像接口明确上传、推理、结果回传和医生确认流程。
- 管理端增加 AI 结果质量评估与人工反馈入口。

## 阶段六：安全、权限与审计

目标：满足医疗业务系统的基本安全要求。

- 完整登录、JWT 或 Session 鉴权。
- 角色权限：患者、医生、医技、药房、财务、管理员。
- 接口级权限校验。
- 敏感数据脱敏：身份证、手机号、病历、处方。
- 操作审计：诊断确认、处方确认、退费、退药、排班干预。
- 配置安全：密钥只允许从环境变量或密钥管理系统读取。

## 推荐分工

- 后端业务组：患者、挂号、排班、缴费、药房、医技核心流程。
- AI 服务组：分诊、病历草稿、处方推荐、排班 Agent、向量检索。
- 前端组：Vue 页面、路由、状态管理、接口对接。
- 数据库与 DevOps：表结构、初始化数据、Docker、CI、部署脚本。
- 测试与质量组：接口测试、并发测试、验收用例、演示数据。
