# 患者端页面外框一致性修复计划

## Goal

让患者端在页面切换时始终使用同一内容宽度、同一桌面端外框和同一底部导航对齐基准，消除页面外缘在圆角与直角之间跳变的问题。

## Current behavior

- `PatientLayout.vue` 已通过 `--patient-page-width` 将患者端主内容限制为 430px，但页面仍各自声明桌面端 `max-width`、阴影或圆角。
- `PatientHomeView.vue` 在桌面端单独使用 28px 圆角；多数其他患者页面没有对应外层圆角，因此在路由切换时外框形态不一致。
- `PatientBottomNav.vue` 以 `100vw` 推导左右边距。`100vw` 包含浏览器纵向滚动条宽度，而页面布局宽度不包含它；不同页面的滚动状态会使导航与主内容产生横向偏移。
- 桌面端页面切换时，是否出现纵向滚动条还会改变可用布局宽度；首页与登录页的患者外壳实测相差约一个滚动条宽度。

## Proposed solution

1. 在 `PatientLayout.vue` 建立唯一的患者端页面壳：显式使用同一宽度，桌面端统一阴影、圆角与 `overflow: hidden` 裁切。
2. 将底部导航改为相对同一壳宽度居中，避免以 `100vw` 作为计算基准。
3. 在根元素稳定预留纵向滚动条槽位，避免页面滚动状态变化导致整个患者端横向跳动。
4. 移除患者页面中重复的桌面端外框样式，保留页面内部卡片和图标所需的语义圆角。
5. 在 `docs/frontend-plan.md` 的维护规则中记录该布局边界，避免后续页面再次引入独立页面外框。
6. 首页与挂号记录页共用 `--patient-flow-page-bg`，使外壳裁切后的页面边缘颜色一致。
7. 首页与挂号记录页共用 `--patient-flow-page-bg` 和患者端外壳；首页保留入口 Hero，挂号记录页采用更紧凑的 230px 页头与 -50px 内容衔接，避免记录页首屏留白过多。

## Risks

- `overflow: hidden` 只能加在桌面端外框，不能影响移动端页面内的弹层、下拉和固定导航。
- 页面内部卡片、头像和图标的圆角是内容语义，不应被全局替换。
- 需避免触碰医生端和管理员端的布局规则。

## Validation strategy

- 运行 `npm run build`。
- 在 390px、430px 与桌面宽度下检查首页、登录、个人中心、消息、挂号流程和候诊页面。
- 在内容较短与可滚动页面之间来回切换，确认主壳宽度、外缘形态和底部导航位置均不跳变。
