# Pull Request 标题

feat(backend): 完善 AI 可信化与核心辅助业务链路

# Pull Request 内容

## 变更摘要

本 PR 汇总后端近期 AI 可信化、收费退费、医生 Agent、排队叫号和医技辅助流程的加固工作。

主要包含：

- AI 可信化：补充 AI 审计日志、脱敏查询、权限控制、数据库事实校验、分诊和处方审核二次验证。
- AI 上下文治理：优化分诊提示词上下文，避免每次携带完整历史对话导致超长上下文。
- AI 结果质量：分诊置信度改为动态计算，医生 Agent 记录工具调用上下文，相似病例返回相似度证据。
- 收费退费可靠性：补充幂等键强制校验、重复收费锁、退费 SAGA 步骤记录、退费恢复保护和迁移脚本。
- 医生 Agent 安全：排班申请改为二阶段确认，失败不再误报成功，重复申请会复用待处理记录。
- 挂号队列修复：今日候诊队列按实际排班日期过滤，并改为按预约时间槽排序。
- 排队叫号接口：新增叫下一位、开始接诊、结束就诊接口，保留旧状态接口兼容原调用。
- 就诊状态闭环：病历确认后 patient 服务能把已挂号患者自动推进到接诊中再结束，避免病历已确认但队列仍显示候诊。
- 退号一致性：退号前会强制查询收费服务，存在非已退费账单时禁止退号。
- 医技辅助流程：开检查/检验/处置前校验挂号状态，新增检验结果录入和处置结果录入接口。
- 队列通知：支付成功、退号、状态切换、叫号等关键队列变化统一触发队列事件。
