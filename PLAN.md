# PLAN

## Goal

实现管理员端最小可运行闭环，优先保证项目完整性、管理员端路由可访问、关键页面可渲染、真实接口可联通，并通过前端构建验证。

## Current Behavior

- 管理员端当前只有登录页和单页占位控制台。
- 路由中只有 `/admin/login` 和 `/admin/console`。
- 后端已经提供排班、审批、账单退费、药房入库/发药/退药、AI 审计等接口，但前端尚未封装和消费。
- 项目没有现成前端测试框架，当前可依赖的验证手段主要是 `vue-tsc --noEmit` 和 `vite build`。

## Proposed Solution

- 先补管理员端专用 API 层，统一封装排班、审批、账单、药房、AI 审计接口。
- 扩展管理员端路由和布局，形成总控台 + 子工作台的导航结构。
- 落地 6 个最小可运行页面：`console`、`schedules`、`approvals`、`audit`、`pharmacy`、`billing`。
- 真实接口优先；后端缺少批量列表接口的区域，采用“明确的操作工作台 + 输入驱动查询/执行”，避免伪造大量假数据。
- 最终通过构建验证保证页面和类型可运行。

## Risks

- 后端接口返回字段较松散，前端字段映射可能需要按真实返回结构调整。
- 药房和账单缺少完备的后台列表接口，一期页面更适合做操作型工作台而不是全量运营看板。
- 项目目前缺少前端单元测试框架，本轮不能做到完整自动化行为测试。

## Validation Strategy

- 对管理员端相关 TypeScript/Vue 文件做静态类型检查。
- 运行 `npm run build` 验证路由、组件、API 封装和页面编译通过。
- 核对关键页面是否都已挂载到路由。
- 汇报未覆盖的真实联调风险点。
